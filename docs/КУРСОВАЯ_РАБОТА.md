# Разработка мобильного приложения "Калькулятор"

## ВВЕДЕНИЕ

### Актуальность темы
В современном мире мобильные приложения стали неотъемлемой частью повседневной жизни. Среди многообразия доступных приложений особое место занимают утилитарные программы, такие как калькуляторы. Несмотря на наличие встроенных калькуляторов в операционных системах, существует потребность в специализированных приложениях с расширенным функционалом, удобным интерфейсом и дополнительными возможностями.

Актуальность разработки мобильного приложения "Калькулятор" обусловлена возрастающими требованиями пользователей к функциональности, удобству использования и визуальному оформлению повседневных инструментов. Современный калькулятор должен не только выполнять базовые арифметические операции, но и предоставлять дополнительные математические функции, сохранять историю вычислений и обладать адаптивным интерфейсом.

### Цель и задачи проекта
**Целью** данной курсовой работы является разработка кроссплатформенного мобильного приложения "Калькулятор" с расширенным функционалом и современным пользовательским интерфейсом.

**Задачи проекта:**
1. Провести анализ существующих решений и технологий разработки
2. Разработать архитектуру приложения
3. Реализовать основной функционал калькулятора
4. Протестировать работоспособность приложения
5. Подготовить документацию по проекту

### Объект и предмет исследования
**Объект исследования:** процесс разработки кроссплатформенных мобильных приложений.

**Предмет исследования:** методы и технологии создания мобильного приложения "Калькулятор" с использованием фреймворка Flutter.

## ГЛАВА 1. АНАЛИТИЧЕСКАЯ ЧАСТЬ

### 1.1. Анализ существующих решений
Рынок мобильных приложений предлагает множество калькуляторов, однако большинство из них имеют ограниченный функционал или перегружены ненужными функциями. Проведенный анализ показал, что существует потребность в сбалансированном решении, сочетающем простоту использования с расширенными математическими возможностями.
### 1.2. Сравнение технологий разработки

#### Мобильная разработка: нативная vs кроссплатформенная

**Нативная разработка** предполагает создание отдельных приложений для каждой платформы:
- **Android:** Java/Kotlin + Android Studio
- **iOS:** Swift + Xcode

**Преимущества нативной разработки:**
- Максимальная производительность
- Прямой доступ к API устройства
- Нативный пользовательский интерфейс

**Недостатки:**
- Высокие затраты на разработку
- Необходимость поддержки двух кодовых баз
- Длительный процесс разработки

**Кроссплатформенная разработка** позволяет создавать приложения для нескольких платформ из единой кодовой базы. Основные технологии:
- **Flutter** (Dart) - выбран для данного проекта
- **React Native** (JavaScript)
- **Xamarin** (C#)

### 1.3. Выбор Flutter для разработки

Flutter был выбран в качестве основного фреймворка по следующим причинам:

**Технические преимущества:**
- Единая кодовая база для iOS и Android
- Высокая производительность благодаря компиляции в нативный код
- Богатая библиотека готовых виджетов
- Горячая перезагрузка (Hot Reload) для ускорения разработки

**Экономические преимущества:**
- Сокращение времени разработки на 30-50%
- Уменьшение затрат на поддержку
- Быстрый вывод продукта на рынок

**Особенности языка Dart:**
- Строгая типизация для надежности кода
- Современный синтаксис
- Высокая производительность
- Полная интеграция с Flutter

### 1.4. Архитектура приложения

Для проекта была выбрана архитектура на основе следующих компонентов:

**Слои приложения:**
1. **Presentation Layer** - пользовательский интерфейс
2. **Business Logic Layer** - обработка математических операций
3. **Data Layer** - хранение истории вычислений

**Основные модули:**
- Калькулятор - ядро приложения
- История вычислений
- Настройки интерфейса
- Система тем

## ГЛАВА 2. ПРАКТИЧЕСКАЯ РЕАЛИЗАЦИЯ

### 2.1. Проектирование интерфейса

Интерфейс приложения был спроектирован с учетом принципов Material Design. Основные требования к UI:
- Интуитивная понятность
- Минималистичный дизайн
- Адаптивность под разные размеры экранов
- Поддержка темной и светлой темы

**Структура экранов:**
- Главный экран калькулятора
- История вычислений
- Настройки приложения

### 2.2. Реализация математического ядра

Основной функционал калькулятора включает:

**Базовые операции:**
- Сложение (+)
- Вычитание (-)
- Умножение (×)
- Деление (÷)

**Дополнительные функции:**
- Проценты (%)
- Смена знака (±)
- Квадратный корень (√)
- Возведение в квадрат (x²)
- Обратное число (1/x)

**Особенности реализации:**
- Обработка деления на ноль
- Корректная работа с десятичными дробями
- Валидация вводимых данных
